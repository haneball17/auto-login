- 当前处于“可见窗口 / 进程存在 / 强制重启”路径  
- 关键重试次数与超时配置  
- 清理动作执行结果

## 8. 风险与边界
- 进程名不匹配会导致清理失败 → 需保证 `launcher_process_name` 准确
- 启动器重跑行为依赖系统状态 → 需有重试和强制重启兜底
- 若游戏窗口出现但启动器清理失败 → 需记录日志并继续流程

## 9. 测试与验证（设计层面）
建议验证路径：
1) 启动器正常启动 → 点击 → 游戏窗口出现 → 启动器进程被清理  
2) 启动器托盘化后进程存在 → 重跑 exe 弹窗 → 正常继续  
3) 重跑 exe 不弹窗 → 强制重启 → 成功弹窗  
4) 强制重启仍失败 → 留证并退出

## 10. 未来扩展（进程复用）
当需要“复用进程”时，可引入策略字段，例如：
- `launcher.lifecycle_mode = clean | reuse`
- `reuse` 下不自动清理进程，仅在失败时重启

此扩展不影响当前设计结构。
